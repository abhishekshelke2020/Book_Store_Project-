# Bookstore Sales Data Analysis (SQL Project)
📌 Project Overview

This project analyzes a bookstore dataset using SQL across three tables: Books, Customers, and Orders.

The objective is to explore sales performance, customer behavior, inventory status, and revenue insights by writing optimized SQL queries.

The dataset includes:

Books.csv – Book details such as title, genre, author, price, stock, and publication year

Customers.csv – Customer information including name, country, and city

Orders.csv – Order transactions including quantity purchased, order date, and total amount

🎯 Key Objectives

In this project, I solved real-world business problems such as:

Total books sold per genre

Average price analysis for specific genres

Identifying top customers and high-value orders

Finding the most frequently ordered and most expensive books

Revenue calculation and stock remaining after orders

Customer location analysis based on spending

Inventory insights including lowest and total stock

🛠 Tools & Skills Used

SQL (Joins, Aggregations, Group By, Having, Subqueries)

Data Analysis

Business Insights Generation

Problem Solving

This project demonstrates how SQL can be used to:

Analyze sales trends

Understand customer purchasing behavior

Track inventory performance

Generate revenue insights

Support data-driven decision making
